# QQQ Advisor 模型规则说明

本文档说明当前模型如何生成投资建议、如何记录历史建议、如何复盘准确率，以及后续应该如何优化。

本项目是投资辅助工具，不是投顾服务。程序输出的是基于规则的分批买入建议，最终下单前必须以支付宝基金页面的申购状态、限购、费率、确认日和赎回规则为准。

## 1. 模型目标

模型的目标不是预测明天涨跌，而是回答三个更实用的问题：

1. 现在是否适合开始买入纳斯达克100相关基金。
2. 如果适合买，本次应该买计划仓位中的多少。
3. 这个建议在 1、5、20 个交易日后看是否有效，从而逐步优化规则。

当前模型适合中长期分批买入，不适合日内交易、短线追涨杀跌或高频择时。

## 2. 数据来源

### 2.1 QQQ 行情

程序优先使用配置了 API key 的日线数据源，然后再使用公开数据源：

1. Twelve Data time series API：需要 `QQQ_TWELVE_DATA_API_KEY`
2. Tiingo daily prices API：需要 `QQQ_TIINGO_API_TOKEN`
3. Investing financialdata historical JSON endpoint：默认 `QQQ_INVESTING_FINANCIALDATA_ID=20`
4. Google Finance 页面数据：默认 `QQQ_GOOGLE_FINANCE_SYMBOL=NDX:INDEXNASDAQ`
5. Nasdaq charting historical JSON endpoint
6. Stooq: `https://stooq.com/q/d/l/?s=qqq.us&i=d`
7. Yahoo Finance chart API: `https://query1.finance.yahoo.com/v8/finance/chart/QQQ`

如果前一个数据源失败，会自动尝试下一个。模型使用最新可得的 QQQ 日收盘价，不使用盘中价格。

QQQ 的成交量会从 Yahoo Finance chart API 一起读取，用于计算：

- `qqq_volume`: 最新成交量
- `qqq_volume_ratio20`: 最新成交量 / 近20日平均成交量
- `qqq_volume_trend20_pct`: 最新成交量相对近20日均量的偏离百分比

### 2.1.1 外部市场因子

程序会额外从 Yahoo Finance chart API 拉取以下日线序列：

| 因子 | Yahoo 符号 | 用途 |
|---|---|---|
| VIX | `^VIX` | 衡量美股波动率和风险偏好 |
| 美债10年收益率 | `^TNX` | 衡量长端利率压力 |
| DXY 美元指数 | `DX-Y.NYB` | 衡量美元强弱和全球流动性环境 |

这些序列会按 QQQ 交易日对齐。如果某个市场当天没有数据，程序会使用该日期之前最新可得的数据；如果仍然缺失，该因子会被跳过，不会让模型报错。

当前写入市场特征的字段包括：

- `vix_close`
- `vix_momentum_20d_pct`
- `us10y_close`
- `us10y_momentum_20d_pct`
- `dxy_close`
- `dxy_momentum_20d_pct`

### 2.2 国内基金净值快照

基金估值使用东方财富公开接口：

```text
https://fundgz.1234567.com.cn/js/{基金代码}.js
```

该接口只是辅助展示，不能替代支付宝交易页。支付宝是否可买、限购、费用和确认规则以支付宝为准。

### 2.3 用户输入

用户在网页端或 `config.json` 中输入：

- `total_investable_cny`: 可投资总金额
- `current_nasdaq_position_cny`: 当前纳指基金持仓金额
- `risk_level`: 风险偏好，支持 `conservative`、`balanced`、`aggressive`
- `horizon_years`: 投资期限
- `max_acceptable_loss_pct`: 最大可接受亏损
- `min_trade_cny`: 单次最低买入金额

## 3. 市场指标

模型每天计算以下指标：

| 指标 | 含义 |
|---|---|
| 最新收盘价 | QQQ 最新交易日收盘价 |
| MA20 | 20 日均线，代表短期趋势 |
| MA60 | 60 日均线，代表中期趋势 |
| MA200 | 200 日均线，代表长期趋势 |
| RSI14 | 14 日相对强弱指标，用来衡量短线是否过热或过冷 |
| 近252日高点回撤 | 当前价格距离近一年高点的跌幅 |
| 20日年化波动率 | 最近20个交易日波动水平 |
| 距离MA20百分比 | 当前价格相对20日均线的偏离程度 |
| QQQ 成交量 | 用成交量放大/缩小识别资金参与强弱 |
| VIX | 波动率和避险压力 |
| 美债10年收益率 | 长端利率变化对成长股估值的压力 |
| DXY | 美元强弱、汇率和全球流动性压力 |
| 宏观事件风险 | 国际局势、政治、经济和大事件的结构化风险摘要 |
| 汇率和流动性 | 美元兑人民币、央行加息/降息、美元流动性松紧 |

## 4. 入场评分规则

模型从 `50` 分开始，根据趋势、回撤、RSI 和波动率加减分，最后限制在 `0-100` 分。

当前评分优先使用 `data/model_params.json` 里的 `factor_model`。旧版 `scoring` 仍保留作为兼容参数。

### 4.0 因子模型

因子定义保存在：

```text
data/model_params.json
```

结构示例：

```json
{
  "factor_model": {
    "enabled": true,
    "factors": {
      "trend_ma200": {
        "enabled": true,
        "type": "above_below",
        "left": "close",
        "right": "ma200",
        "above_points": 12,
        "below_points": -20,
        "weight": 1.0,
        "min_weight": 0.5,
        "max_weight": 1.5
      }
    }
  }
}
```

每个因子的最终贡献：

```text
因子贡献 = 原始分数 * weight
```

支持的因子类型：

| 类型 | 用途 |
|---|---|
| `above_below` | 比较两个字段，例如 `close > ma200` |
| `threshold` | 单字段阈值，例如 `rsi14 >= 75` |
| `range` | 区间判断，例如 `68 <= rsi14 < 75` |
| `threshold_tiers` | 分层阈值，例如不同回撤给不同分 |
| `all_conditions` | 多条件同时成立 |

增删因子：

- 新增：在 `factor_model.factors` 下增加一个新 key。
- 删除：直接删除该 key，或把 `enabled` 改成 `false`。
- 调权重：修改 `weight`。
- 控制探索范围：设置 `min_weight` 和 `max_weight`。

建议不要让单个因子的 `max_weight` 太大，否则模型容易过拟合。

新增的 VIX、美债10年收益率、DXY、QQQ 成交量因子默认先作为候选因子保存，未通过训练验证前不直接影响每日评分。Agent 可以在训练中启用、关闭或调权，但只有验证集表现提升时才会写入生效参数。

### 4.1 趋势分

| 条件 | 分数 |
|---|---:|
| 收盘价高于 MA200 | `+12` |
| 收盘价低于或等于 MA200 | `-20` |
| 收盘价高于 MA60 | `+8` |
| 收盘价低于或等于 MA60 | `-6` |

设计思路：长期趋势没有破坏时，允许分批参与；跌破长期趋势时，降低买入积极性。

### 4.2 回撤分

| 条件 | 分数 |
|---|---:|
| 距近一年高点回撤 `<= -8%` | `+22` |
| 回撤 `<= -5%` | `+15` |
| 回撤 `<= -3%` | `+8` |
| 回撤 `> -1%` 且价格高于 MA20 超过 `4%` | `-18` |

设计思路：纳指长期向上时，回撤通常比追高更适合买入；如果几乎没有回撤且短线偏离 MA20 过大，降低入场分。

### 4.3 RSI 分

| 条件 | 分数 |
|---|---:|
| RSI14 `>= 75` | `-22` |
| RSI14 `>= 68` | `-12` |
| RSI14 `<= 35` 且价格高于 MA200 | `+15` |

设计思路：RSI 很高说明短线可能过热，不适合重仓追入；RSI 很低但长期趋势仍在时，可能是更好的分批买点。

### 4.4 波动率分

| 条件 | 分数 |
|---|---:|
| 20日年化波动率 `> 35%` | `-8` |

设计思路：波动明显放大时，单次买入比例应更谨慎。

## 5. 动作映射

评分会映射成四种动作：

| 评分区间 | 动作 |
|---:|---|
| `>= 78` | `加仓` |
| `>= 62` | `正常买入` |
| `>= 45` | `小额试探` |
| `< 45` | `暂不买入` |

这四个动作只决定“本次是否买、买多少”，不代表卖出建议。

## 6. 仓位和买入金额

### 6.1 目标纳指仓位

基础目标仓位由风险偏好决定：

| 风险偏好 | 基础目标仓位 |
|---|---:|
| `conservative` | `12%` |
| `balanced` | `22%` |
| `aggressive` | `32%` |

然后根据投资期限和可接受亏损调整：

| 条件 | 调整 |
|---|---:|
| 投资期限 `< 1.5` 年 | 乘以 `0.55` |
| 投资期限 `>= 5` 年 | 乘以 `1.15` |
| 最大可接受亏损 `<= 10%` | 乘以 `0.65` |
| 最大可接受亏损 `>= 30%` | 乘以 `1.10` |

最终目标纳指仓位被限制在 `5%-40%` 之间。

### 6.2 本次买入比例

动作对应买入剩余计划金额的比例：

| 动作 | 买入比例 |
|---|---:|
| `加仓` | `35%` |
| `正常买入` | `25%` |
| `小额试探` | `12%` |
| `暂不买入` | `0%` |

计算方式：

```text
目标纳指金额 = 可投资总金额 * 目标纳指仓位
剩余计划金额 = max(目标纳指金额 - 当前纳指持仓, 0)
本次建议买入 = 剩余计划金额 * 动作买入比例
```

如果本次建议买入金额小于 `min_trade_cny`，则置为 `0`。

## 7. 建议记录

每次生成建议时，程序会写入：

```text
data/advice_log.json
```

记录内容包括：

- 模型版本
- 生成时间
- QQQ 对应交易日
- QQQ 收盘价
- 入场评分
- 动作
- 建议买入金额
- 当时的用户资产和风险配置
- 当时的市场指标
- 当时的基金候选池

同一个 QQQ 交易日只保留一条记录。如果当天多次运行，程序会更新这一条记录。

## 7.1 参数文件

模型参数不再只写死在代码里，运行时会读取：

```text
data/model_params.json
```

如果该文件不存在，程序会自动写入默认参数。这个文件包含：

- 目标仓位参数
- 风险偏好调整参数
- 评分加减分参数
- 动作阈值
- 每种动作对应的买入比例
- 自动进化所需的最小样本数

这样 agent 可以在不改 Python 代码的情况下更新模型参数。

## 7.2 支付宝 T+1 执行滞后

模型按支付宝基金 `T+1` 估算确认/成交净值：

```text
下单日 = 今天
估算确认/净值日 = 今天后的第 1 个工作日
```

如果遇到周末、境内外节假日、基金暂停申购或支付宝页面公告，实际确认日可能顺延。最终以支付宝页面为准。

T+1 会带来一个问题：今天看到的 QQQ 收盘信号，不一定等于实际确认时的市场状态。因此模型新增了 `execution_lag` 参数：

```json
{
  "alipay_fund_nav_lag_trading_days": 1,
  "high_volatility_pct": 25,
  "high_volatility_penalty": -3
}
```

当近期波动率较高时，T+1 滞后会额外降低入场评分，避免在高波动环境里用过大的单次金额追入。

## 7.3 宏观事件和国际局势

宏观事件保存在本地 JSON：

```text
data/macro_context.json
```

结构示例：

```json
{
  "updated_at": "2026-05-10",
  "source": "manual_local_json",
  "events": [
    {
      "category": "geopolitics",
      "direction": "risk_off",
      "severity": 4,
      "summary": "某重大地缘冲突升级，市场风险偏好下降",
      "expires_at": "2026-05-17"
    }
  ]
}
```

字段说明：

| 字段 | 含义 |
|---|---|
| `category` | 事件类别，如 `geopolitics`、`war`、`monetary_policy`、`inflation`、`recession`、`trade`、`regulation` |
| `direction` | 对风险资产方向，支持 `risk_off`、`risk_on`、`neutral` |
| `severity` | 严重度，`0-5` |
| `summary` | 简短摘要，不保存长新闻全文 |
| `expires_at` | 事件过期日，过期后不参与评分 |

宏观事件会通过 `macro_scoring` 调整入场评分：

- `risk_off` 事件降低评分。
- `risk_on` 事件提高评分。
- 单轮宏观调整默认最多不超过 `±12` 分。

这部分用于控制仓位和入场节奏，不用于做确定性预测。

汇率和央行流动性通过 `currency_macro_scoring` 进入评分：

- 人民币快速贬值：对人民币投资者的美元资产有汇兑收益可能，但通常也代表风险偏好下降，当前模型小幅扣分。
- 人民币快速升值：降低美元资产换回人民币收益，但通常风险压力较低，当前模型小幅加分。
- 全球央行偏加息：成长股估值承压，扣分。
- 全球央行偏降息：流动性改善，加分。
- 美元流动性收紧：扣分。
- 美元流动性宽松：加分。

这些字段都保存在 `data/macro_context.json`，不需要传长新闻给大模型。

## 7.4 卖出建议

模型现在同时输出：

- `buy_now_cny`
- `sell_now_cny`

卖出不是清仓逻辑，而是“超目标仓位 + 高风险/高估短线状态”下的减仓建议。

默认触发条件：

1. 当前纳指持仓高于目标纳指仓位。
2. 市场评分低于卖出阈值，或 RSI 很高且接近一年高点。
3. 单次卖出不超过当前纳指持仓的一定比例。

卖出建议也只会生成待手动确认交易单，不会自动提交支付宝。

## 7.5 每日建议和真实成交分离

每日建议保存在：

```text
data/advice_log.json
```

待手动确认交易单保存在：

```text
data/manual_orders.json
```

真实成交记录保存在：

```text
data/actual_trades.json
```

复盘时先评估建议本身的准确度，再单独评估真实成交结果。这样可以区分：

- 模型判断是否正确。
- 用户是否执行。
- 执行时点和金额是否影响最终收益。

## 8. 复盘规则

命令行复盘：

```bash
python3 qqq_advisor.py evaluate
```

网页端会自动调用复盘 API：

```text
GET /api/evaluation
```

当前复盘窗口：

- `1d`: 1 个交易日后
- `5d`: 5 个交易日后
- `20d`: 20 个交易日后

判断规则：

| 当时动作 | 后续表现 | 结果 |
|---|---|---|
| `小额试探`、`正常买入`、`加仓` | 后续 QQQ 收益 `> 0` | 正确 |
| `小额试探`、`正常买入`、`加仓` | 后续 QQQ 收益 `<= 0` | 错误 |
| `暂不买入` | 后续 QQQ 收益 `<= 0` | 正确 |
| `暂不买入` | 后续 QQQ 收益 `> 0` | 错误 |

这个复盘规则很直观，但比较粗糙。它用来发现模型偏差，不应该单独作为投资成败的完整评价。

## 9. 优化提示

当前模型不会自动改参数，只会根据复盘样本给出提示。

样本不足时：

```text
复盘样本还少，先连续记录至少 5 个具备20交易日结果的建议，再调整模型参数。
```

当至少有 5 条具备 20 交易日结果的样本后，会检查：

| 情况 | 提示 |
|---|---|
| 多次 `暂不买入` 后 20 日涨幅超过 `3%`，且当时 RSI >= 70 | 可能过度惩罚强趋势高 RSI |
| 多次建议买入后 20 日跌幅超过 `3%` | 可能买入门槛太低，或分批比例太高 |
| 没有明显偏差 | 继续积累样本 |

后续优化建议优先按这个顺序做：

1. 至少积累 20-30 条记录后再改参数。
2. 先调整阈值和分数，不要急着引入复杂机器学习。
3. 每次只改一个方向，例如 RSI 扣分或回撤加分。
4. 修改后提升 `MODEL_VERSION`，方便区分不同版本表现。
5. 同时观察 5 日和 20 日结果，不只看 1 日。

## 10. 当前局限

1. 使用 QQQ 作为纳斯达克100代理，没有直接使用人民币 QDII 基金实际成交净值复盘。
2. 暂未计入汇率变化、QDII 滞后确认、申购赎回费、限购和节假日。
3. 暂未提供卖出、止盈、止损模型。
4. 复盘样本少时准确率没有统计意义。
5. 免费公开行情源可能偶尔失败或延迟。

## 11. 推荐迭代方向

短期：

1. 每天固定时间自动运行并记录建议。
2. 手动在支付宝确认是否真的下单，并把实际成交记录补入系统。
3. 增加“是否执行建议”的字段，区分模型建议和真实交易。

中期：

1. 引入基金实际净值复盘，而不只看 QQQ。
2. 加入美元人民币汇率。
3. 加入交易费用估算。
4. 增加回撤分层定投计划。

长期：

1. 建立参数回测工具。
2. 对不同模型版本做对比。
3. 引入宏观利率、估值、盈利预期等因子。

## 12. Agent 自动进化

自动进化入口见：

[AGENT.md](/Users/dz0401012/Desktop/pythonProject/qqq/AGENT.md)

核心原则：

1. Agent 可以提出候选参数，但不能绕过本地回测。
2. Qwen 可以参与分析，但不能直接决定投资动作或直接改代码。
3. 所有参数、日志、状态都保存在本地 JSON 文件中。
4. 样本不足时只记录和观察，不自动进化。

### 12.1 因子权重自探索

Agent 会在本地做小步探索：

1. 读取当前 `factor_model.factors`。
2. 对启用的因子尝试 `weight + 0.1` 和 `weight - 0.1`。
3. 不超过每个因子的 `min_weight` 和 `max_weight`。
4. 用历史建议做 20 个交易日复盘。
5. 只有候选权重组合比当前参数表现更好，才会写入 `data/model_params.json`。

这意味着模型可以逐步探索：

- 哪些因子应该更重要。
- 哪些因子应该弱化。
- 某些因子是否应该关闭。

当前版本不会自动创造完全未知的新因子，因为新增因子需要明确字段含义。你可以手动新增因子定义，Agent 后续会探索它的权重。

## 13. 大模型 Token 精简策略

传给 Qwen 的内容必须压缩：

1. 不传长新闻正文。
2. 不传完整 `advice_log.json`。
